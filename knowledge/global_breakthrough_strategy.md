# 全局破局策略 — 6 环境关键突破点

> 日期: 2026-03-18 | 目标: Affine Leaderboard #1
> 当前估算 GM ≈ 31.5 | 榜首估算 GM ≈ 52.7 | 差距: ~21 点

---

## 一、核心机制决定战略框架

排行榜算法的三个核心特性决定了一切决策：

1. **几何均值 = 木桶效应** — 最弱环境对 GM 的拖累远大于最强环境的贡献
2. **L6 权重 192 vs L1 权重 6** — 全环境覆盖的子集权重是单环境的 32 倍
3. **排名衰减 0.5** — 在任一子集中 rank 2→rank 1 的收益 = 翻倍；均衡 > 偏科

**一句话战略**: 把最弱的两个环境（NAVWORLD 5.7、GAME 22.6）拉上来，比把最强的环境（LGC-v2 95）推更高，对 GM 的贡献大 10 倍。

## 二、6 环境破局关键一览

### 环境 1: NAVWORLD — 最大杠杆点 (5.7 → 25+)

| 维度 | 现状 |
|------|------|
| 分数 | 5.7 (28% non-zero) |
| 榜首 | 33.7 (RLStepone, RL 方法) |
| GM 贡献 | 5.7→20 = **+7.2 GM**，所有环境中最高 |

**为什么卡住**: SFT 数据量瓶颈已确认 — +240% 数据仅 +12% 提升。全员弱势（7-34分），说明评分机制本身有高门槛。

**破局关键**:
- **质量 > 数量**: 现有 2,248 条标注均为 score=1.0（自动生成），实际质量参差。需用评分器重新筛选，留 ~800 条高质量
- **DPO 是必经之路**: 241 对偏好对已就绪。SFT 天花板约 15-20，DPO 可推至 25-30
- **评分解构**: 50 分代码评分（工具多样性 ≥3 种、方案 ≥800 字、POI grounding）+ 50 分 LLM 语义评分。针对性优化这些指标
- **部署注意**: sglang 必须加 `--tool-call-parser qwen25`，否则 0 分

**行动优先级**: P0 — 每提升 1 分对 GM 的贡献最大

### 环境 2: GAME — 结构性天花板需方法切换 (22.6 → 40+)

| 维度 | 现状 |
|------|------|
| 分数 | 22.6 (39% non-zero) |
| 榜首 | 63.2 (RLStepone, RL 方法) |
| GM 贡献 | 22.6→35 = **+3.1 GM** |

**为什么卡住**: 22 个游戏中仅 ~9 个 SFT 可学，othello/hex/go/chess 等需搜索深度的游戏永远 0%。SFT 天花板 ~40-50。

**破局关键**:
- **补齐缺失游戏**: hearts、bridge、blackjack、euchre 4 个 Strong-tier 游戏完全没有数据，补上可将 non-zero rate 39%→50%+
- **Bot 策略数据有效**: gin_rummy 加入 bot 数据后从 0%→100%，证明路径可行
- **DPO 突破 SFT 天花板**: 589 对偏好对已就绪，优先用于可学习但不稳定的游戏
- **放弃不可学游戏**: go、chess、checkers、solitaire 不投入任何资源

**行动优先级**: P1 — 补 bot 数据是低成本高回报，DPO 是中期必做

### 环境 3: SWE-SYNTH — 序列长度是唯一瓶颈 (31 → 38+)

| 维度 | 现状 |
|------|------|
| 分数 | ~31 |
| 榜首 | ~44 (AnastasiaFantasy) |
| GM 贡献 | 31→38 = **+1.2 GM** |

**为什么卡住**: 98% 数据在 seq=4096 时被截断。模型只学到对话开头，从未见过完整修复过程。

**破局关键**:
- **seq=8192 是免费午餐**: 仅改训练参数，可用数据从 ~2% → ~30%（约 400 条完整对话）
- **清除 think tag 污染**: 24.7%（334 条）含 `<think>` 标签，与 THOUGHT 前缀格式冲突，必须清除
- **无法本地验证**: 只能通过部署后观察排行榜分数
- **DPO 备选**: 258 对偏好对可用

**行动优先级**: P0 — 改一个训练参数即可获得收益，成本最低

---

### 环境 4: LIVEWEB — 当前无解，维持非零 (24 → 24)

| 维度 | 现状 |
|------|------|
| 分数 | ~24 |
| 榜首 | ~28 |
| GM 贡献 | 24→28 = **+1.0 GM**，但需极高成本 |

**为什么卡住**: 数据长度灾难 — 中位 145K chars（~36K tokens），即使 seq=8192 仍然 0% 数据可完整容纳。合成数据全部失败。

**破局关键**:
- **需要上游改造**（修改 liveweb-arena 源码）: DOM 压缩 + 页面去重 + 历史压缩 → 39K→8-10K tokens
- **不改上游 = 无法提升**: 这是硬性约束，不是数据或方法问题
- **全员差距小**: 16-28 分区间，差异极小，说明任务本身对 LLM 很难
- **安全网策略**: 保持现有 10 条短数据确保非零即可

**行动优先级**: P3 — ROI 最低，除非用户授权上游修改

## 四、关键认知总结
