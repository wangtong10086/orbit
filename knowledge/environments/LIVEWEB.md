# LIVEWEB Environment

## Key Facts
- Web interaction/browsing evaluation via liveweb-arena Docker container
- Format: OpenAI function calling (tool_calls with goto/click/type/stop etc.)
- Eval uses LLM validator to compare agent answers vs ground truth
- **5-step observation window**: agent only sees last 5 steps in "Recent Actions"
- Each step is independent LLM call (system + user), NOT multi-turn conversation

## Eval Architecture
- **Agent**: receives accessibility tree + task prompt, outputs tool_calls
- **Actions**: goto, click, click_role, type, type_role, press, scroll, view_more, wait, stop
- **Scoring**: agent answer vs ground truth via LLM validator (0.0-1.0), geometric mean across subtasks
- **Plugins (active)**: coingecko (8), taostats (10), stooq (7+), hackernews (4), hybrid (8)
- **Not covered by teacher**: openlibrary (4), arxiv (5), openmeteo (4) — cache incomplete
- **Disabled**: weather (6 templates)

```bash

cd /root/liveweb-arena-teacher

# Single-task (all templates × 50 seeds)
python3 scripts/teacher_generate.py --output-dir out_s1 --seeds 1-50

# Multi-subtask
python3 scripts/teacher_generate.py --output-dir out_s2 --seeds 1-1000 --num-subtasks 2

```

## Think Chain Design

Each `<thinking>` block contains structured reasoning:

**Step 1 (planning):**
```
I need to answer: "What is Bitcoin's market cap?"
This requires visiting one page and extracting the answer directly.
Next: https://www.coingecko.com/en/coins/bitcoin
Reason: Visit Bitcoin detail page
```

**Data extraction step (citing tree):**
```
Current page: Bitcoin detail page
I can see on the page:
  "18 Mar, 21:02  198.6100  +0.6400 (+0.32%)"
Extracted: close = 198.61

=== Working Memory ===
  Bitcoin: close=198.61
  Visited: 1/3 pages
  Still need: 2 more page(s)
=====================
```

**Final step (computation):**
```
All data collected:
  Chevron: close=198.61, high=200.73, low=198.51
  GE Aerospace: close=286.79, high=293.43, low=282.29
Comparing: GE Aerospace has largest (high-low)/close
Answer: GE Aerospace
```

## Quality Characteristics

**Strong points:**
- stooq/coingecko steps: precise tree references ("I can see on the page: ...")
- Working Memory accumulates across steps correctly
- Multi-subtask: cross-site navigation (stooq→coingecko→taostats)
- Zero format issues, zero bad answers

**Known limitations:**
- Only goto+stop actions — no click/type/scroll (eval may need these)
- HN/taostats aggregation steps: thinking says "page contains data" without listing specifics
- 2-subtask is eval default but only 24% of data (could be higher)

## Training Format
- Canonical `data/canonical/liveweb.jsonl` stores OpenAI tool_calls format
- `forge rental prepare-data` converts to Qwen3 native `<tool_call>` XML
- `content=None` → `""` auto-fix in `canonical_ops.py`
- All entries fit seq_len=8192
