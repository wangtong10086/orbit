# 评测 0 分问题分析

## NW 0 分 — 根因：ms-swift strip tool_calls

### 问题
ms-swift `_check_messages` 将 assistant 消息的 `tool_calls` 字段删除（白名单只保留 role/content/loss）。NW 数据 10006 条全部使用 OpenAI `tool_calls` 格式，strip 后 assistant content="" → 模型学到空回复 → eval 时不调用工具 → 0 分。

### 修复
用 `scripts/convert_openai_to_msswift.py` 将 NW 的 `tool_calls` → `role: "tool_call"`, `role: "tool"` → `role: "tool_response"`。已验证 token 输出完全一致（6830 tokens）。

### 影响环境
- NW: 10006 条全部受影响
- LW（新版 19776 条）: 已用 3-msg 格式重做，无 tool_calls，不受影响
- GAME/MG/SWE-I: 无 tool_calls，不受影响

## SWE-SYNTH 0 分 — 根因：eval 环境缺预生成任务

### 问题
SWE-SYNTH eval 环境要求 task 必须由 breaker service 预先生成。eval 脚本生成随机 task ID（`rng.randint(0, 2^31-1)`），这些 ID 在 SWE-SYNTH 环境中不存在。

错误：`Task xxx not found. Tasks must be pre-generated by the breaker service.`

100/100 全部是环境 error，模型没有机会回答。

### 修复方案
1. **添加 `cached_task_ids`**: 像 LIVEWEB 一样，提供一个预生成的 task ID 列表文件
2. **运行 breaker service**: 在 eval 前先跑 `breaker/service.py` 生成任务
3. **用 SWE-INFINITE 替代**: 检查 SWE-INFINITE 环境是否不需要预生成

### SWE-INFINITE vs SWE-SYNTH
eval_envs.py 只配置了 SWE-SYNTH，但 affinetes 目录下有 SWE-INFINITE 环境。可能 SWE-INFINITE 是新的评测环境，需要确认。
